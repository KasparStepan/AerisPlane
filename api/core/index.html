    Core — aerisplane.core¶

    The core module contains the geometry and component dataclasses used by every discipline module. It depends only on numpy — no solvers or backends.

    Coordinate system¶

    All positions use the aircraft body frame:

    • x — positive aft (toward the tail)
    • y — positive right (starboard wing)
    • z — positive up

    xyz_le on WingXSec and Wing is the leading-edge position in this frame. For symmetric wings, define the right side (y ≥ 0) only; the left side is mirrored automatically.

    Return the first wing that looks like a main wing (largest area).

    Wing and WingXSec¶

    aerisplane.core.wing.Wing dataclass ¶

    A lifting surface composed of spanwise cross-sections.

    Used for main wings, horizontal tail, and vertical tail — there is no separate Tail class. A horizontal stabilizer is just a Wing with a different name and position.

    Parameters:

    Name Type Description Default
    name str

    Descriptive name (e.g., "main_wing", "htail", "vtail").

    		 'wing'
    xsecs list of WingXSec

    Ordered from root (y=0 for symmetric) to tip.

    		 list()
    symmetric bool

    If True, the wing is mirrored about the xz-plane (y=0).

    		 True
    control_surfaces list of ControlSurface

    Control surfaces on this wing.

    		 list()

    color = None class-attribute instance-attribute ¶

    Optional display colour for visualization. Any CSS colour string, e.g. "#FF4444".

    If None, the visualization module picks from its default palette.

    semispan() ¶

    Semispan: distance from root to tip along y-axis [m].

    For a non-symmetric wing, this is the full extent from first to last xsec.

    span() ¶

    Total wingspan [m].

    For symmetric wings: 2 * semispan. For non-symmetric wings (e.g., vertical tail): semispan.

    area(type='planform') ¶

    Wing area [m^2].

    Parameters:

    Name Type Description Default
    type str

    "planform" (default): trapezoidal planform area. "wetted": surface area including airfoil thickness, both surfaces. "xy": planform area projected onto XY plane (top-down view). "xz": area projected onto XZ plane (side view).

    		 'planform'
    For
    		required

    aspect_ratio() ¶

    Aspect ratio AR = b^2 / S.

    mean_aerodynamic_chord() ¶

    Mean aerodynamic chord by integration: MAC = (1/S_semi) * integral(c^2 dy) [m].

    Reference: Raymer, Aircraft Design, Ch. 7.

    mean_aerodynamic_chord_le() ¶

    Leading-edge position of the MAC [x, y, z] in aircraft frame [m].

    Computed as the area-weighted average of section LE positions.

    aerodynamic_center(chord_fraction=0.25) ¶

    Aerodynamic center [x, y, z] in aircraft frame [m].

    Uses exact per-section MAC formula (accounts for taper ratio per section), then area-weights the result across all sections.

    Parameters:

    Name Type Description Default
    chord_fraction float

    Position along the MAC as a fraction of chord length (default 0.25).

    		 0.25

    taper_ratio() ¶

    Taper ratio: tip chord / root chord.

    sweep_le() ¶

    Leading-edge sweep angle [deg].

    Computed from root and tip LE x-positions.

    dihedral() ¶

    Geometric dihedral angle [deg].

    Computed from root and tip LE z-positions.

    mean_sweep_angle() ¶

    Mean quarter-chord sweep angle [deg].

    Sweep of the line connecting root and tip quarter-chord points, measured from the y-axis in the xy-plane.

    mean_dihedral_angle() ¶

    Mean dihedral angle [deg].

    Dihedral of the line connecting root and tip LE points, in the yz-plane.

    mean_geometric_chord() ¶

    Mean geometric chord: area / span [m].

    mean_twist_angle() ¶

    Area-weighted mean twist angle [deg].

    Positive = nose-up. Zero for an untwisted wing.

    volume() ¶

    Total structural volume of the wing [m^3].

    Uses the Prismatoid formula per section (exact for frustum-shaped bodies). Includes both sides for symmetric wings.

    get_control_surface_names() ¶

    Names of all control surfaces defined on this wing.

    control_surface_area(by_name=None) ¶

    Total planform area of control surfaces [m^2].

    Parameters:

    Name Type Description Default
    by_name str or None

    If given, only count the control surface with this name.

    		 None

    is_entirely_symmetric() ¶

    True if wing geometry is symmetric AND all control surfaces are symmetric.

    translate(xyz) ¶

    Return a copy of this wing translated by xyz [m].

    draw(backend='plotly', show=True, **kwargs) ¶

    Visualize this wing. See aerisplane.viz.draw for full docs.

    sectional_span_yz() ¶

    Span of each section measured in the YZ plane [m].

    Returns one value per section (len(xsecs) - 1 entries).

    sectional_areas() ¶

    Planform area of each section [m^2].

    Computed as the average chord times the YZ-plane section span. Returns one value per section (len(xsecs) - 1 entries).

    sectional_aerodynamic_centers() ¶

    Approximate aerodynamic center of each section [x, y, z] in metres.

    Computed as the midpoint of the section's quarter-chord line. Returns one (3,) array per section (len(xsecs) - 1 entries).

    Internal volume [m^3].

    Uses the Prismatoid formula per section, which is exact for frustums (tapered cylinders) and more accurate than trapz for nose/tail cones:

    V_section = (x_b - x_a) / 3 * (A_a + A_b + sqrt(A_a * A_b))

    aerisplane.core.fuselage.FuselageXSec dataclass ¶

    A cross-section of a fuselage at a given axial station.

    Parameters:

    Name Type Description Default
    x float

    Axial position from nose [m]. 0 = nose tip.

    		 required
    width float

    Cross-section width in the y-direction [m].

    		 0.0
    height float

    Cross-section height in the z-direction [m].

    		 0.0
    shape float

    Superellipse exponent. Controls cross-section shape: shape=1.0 → diamond (45° rotated square) shape=2.0 → circle / ellipse (default) shape=5.0 → rounded rectangle shape→∞ → rectangle Use 2.0 for aerodynamic bodies, 4–8 for fuselages with flat sides.

    		 2.0
    radius float or None

    Backward-compat convenience: sets width = height = 2 * radius, shape = 2.0. Use width/height directly for non-circular cross-sections.

    		 None

    area() ¶

    Cross-sectional area [m^2].

    Uses closed-form superellipse approximation (error < 0.6% for shape >= 1):

    area = width * height / (shape^-1.8718 + 1)

    Exact for shape=1 (diamond) and shape=2 (circle/ellipse).

    perimeter() ¶

    Cross-section perimeter [m].

    Closed-form approximation derived by symbolic regression (error < 0.2%).

    equivalent_radius(preserve='area') ¶

    Equivalent circular radius [m] that preserves area or perimeter.

    Parameters:

    Name Type Description Default
    preserve str

    "area" : radius = sqrt(area / pi) "perimeter" : radius = perimeter / (2 * pi)

    		 'area'

    Airfoil¶

    aerisplane.core.airfoil.Airfoil dataclass ¶

    Airfoil defined by name and/or coordinate array.

    If name is given and coordinates is None, the catalog is searched for a matching .dat file. NACA 4-digit names (e.g. "naca2412") are generated analytically if not found in the catalog.

    Parameters:

    Name Type Description Default
    name str

    Airfoil name, e.g. "ag35", "naca2412", "e387". Used for catalog lookup and plot labels.

    		 required
    coordinates ndarray of shape (N, 2) or None

    Explicit (x, y) coordinate array in Selig format (upper surface first, x from 0 to 1 and back to 0). If None, loaded from catalog by name.

    		 None

    Examples:

    >>> af = Airfoil(name="ag35")         # loads from catalog
>>> af = Airfoil(name="naca2412")     # generated analytically

    convert_axes(x_from, y_from, z_from, from_axes, to_axes) ¶

    Convert a vector between geometry / body / wind / stability axes.

    Follows the conventions in Drela, Flight Vehicle Aerodynamics, §6.2.2.

    Geometry axes: x downstream, y right, z up. Body axes: x forward, y right, z down. Wind axes: x into wind, y right, z "up" (lift direction). Stability axes: body axes rotated by alpha about y.
